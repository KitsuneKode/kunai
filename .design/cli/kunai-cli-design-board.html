Kunai CLI design board

Kunai is a user-facing terminal media app. It searches media, resolves direct provider streams, hands playback to mpv, tracks progress, supports recommendations, release calendar, playback recovery, tracks/source selection, watch history, stats, and offline/download flows. The CLI should feel like a premium streaming product translated into a TUI: calm, fast, rich only when it helps a decision, and never like provider logs or a developer dashboard.

Color Roles

Recommended direction: Ember + Teal with clearer state hierarchy

The screen should not be amber plus beige. Use amber only for focus/action, teal for active process and input, green for playable success, blue for upcoming/info, violet for secondary metadata categories, red for real failure.

Stats

Four Required States

Loading

Specific, calm, stable. Example: resolving stream, loading poster, building watch stats. Do not show noisy logs.

Success

Quiet unless meaningful. Example: episode complete, subtitle attached, source ready, download ready.

Empty

Short next action. Example: no stats yet, no recommendations, no library items. No tutorial wall.

Error

Actionable and scoped. Example: stream did not start, provider failed, poster unavailable. Offer recover/fallback/diagnostics.

Anti-Patterns To Avoid

Do not make amber universal.
Amber is focus/action. If everything is amber or beige, hierarchy collapses.
Do not expose provider internals by default.
Provider/cache/subtitle counts go to preview, details, or diagnostics unless they affect the immediate decision.
Do not make every value a badge.
Use text rhythm, selected rows, muted metadata, and preview facts first.
Do not use boxes everywhere.
Prefer spacing and dividers. Boxes are for preview rail, details sheet, real framed tools, and important state groups.
Do not make modals universal portals.
Subpanels stay scoped. Return to previous screen for broader navigation.
Do not let images cause layout jumps.
Reserve poster slots and anchor metadata below the reserved area.
