S4 Daily Surface: Scoped Command Palette

The command palette should feel powerful without becoming a portal out of the current task. Scope is the UX: post-play sees post-play actions, a picker sees picker actions, global appears only when explicitly expanded.

PPS scope
Next, replay, episodes, tracks, search
Commands keep the user inside the post-playback decision. Global actions are hidden behind "more".
Picker scope
Open, filter, sort, close
Picker commands do not jump to unrelated surfaces. Escape always returns to the parent context.
Global scope
Settings, history, diagnostics
Global commands exist, but must not be the default list while the user is making a local choice.

S3 Portability And Stability

This is the work that makes the design feel professional across terminals. It is not glamorous, but it prevents broken layouts, flicker, and image-related jumping.

Poster fallback
image → letter tile → hidden
Always reserve the poster slot. Metadata anchors below it so late images do not move text.
Color fallback
truecolor → 256 → 16
Centralize terminal color adaptation. Screens consume semantic roles only.
Responsive fallback
rail collapses first
Narrow terminal keeps header, main list, active state, and commands. Preview is optional.

Color Strategy Still Missing

S1 made names semantic, but hierarchy is still weak if most text is muted and amber. The next pass should tune where colors appear, not add more colors.

bgnever pure black
surfacesubtle depth
texttitles by weight
mutedmetadata
rosefocus/action
mintplayable/done
amberwaiting/countdown
crimsonreal failure

Rule: color should label state and focus. Hierarchy comes from text weight, row rhythm, spacing, and stable layout. Do not make every title blue, every source teal, or every metadata fact a badge.

Implementation Slices After The Board

01 Shared primitives first
  • Footer action model and collapse rules.
  • StateBlock for loading, empty, error, success.
  • PreviewRail with stable poster reservation.
  • MediaList, ActionList, PickerSurface, CapabilityRows.
02 Recovery surfaces
  • Playback did not start, stream stalled, no source, provider degraded.
  • Recover/fallback/sources/diagnostics are explicit and scoped.
  • Never mark watched on unresolved failure.
03 Daily surfaces
  • Search query vs result-filter separation.
  • Tracks capability normalization.
  • Scoped command palette.
  • DetailsSheet for rich media information.
04 Return loop and portability
  • Browse ready-for-you-now, Calendar new-since-current, History resume.
  • Poster and color fallback, narrow terminal behavior, CJK truncation.
  • Targeted reducer/view-model tests before visual wiring.
