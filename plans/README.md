# Implementation Plans — Kunai production-readiness track

Generated by the improve skill on 2026-07-16, planned against commit `4b351cb0`.

These plans take Kunai from "mostly working with sharp edges" toward a clean,
production-level state: fix the correctness/security bugs on the untrusted-provider
→ mpv boundary, make startup and stream-resolution feel fast, split the god files
behind a test safety net, make the documented architecture real, and trim the
doc/plan surface so the code is self-explanatory. **Videasy provider internals are
intentionally deferred** (per the maintainer) — plan 016 explicitly leaves videasy's
crypto path alone; everything else is the harness _around_ the providers, the TUI
engine, and playback controls.

Each executor: read the plan fully before starting, run the drift check, honor the
STOP conditions, and update your row below when done.

> Note: this `plans/` directory (advisor handoff plans) is separate from the repo's
> own `.plans/` tracker. Plan 017 cleans up `.plans/`; do not conflate the two.

## Findings considered and rejected

Recorded so they aren't re-audited:

- **logs.txt leaking tokens / unbounded growth** — not a finding: the file sink is
  unimplemented, console output is redacted (`redactDiagnosticValue`), and the trace
  reporter redacts before append. (The _missing_ file sink is addressed as a DX item in
  plan 004, not as a security issue.)
- **Playback loop abort-listener accumulation** — not a finding: the per-iteration
  `addEventListener(..., { once: true })` is paired with a `removeEventListener` in the
  `finally` that every loop exit passes through; the recover loop is bounded by
  `MAX_AUTO_SOURCE_RECOVER_ATTEMPTS`. (The re-entrant crash handlers in plan 001 are the
  more likely amplifier of the historical RSS blowup.)
- **Root `logs.txt` / `stream_cache.json` committed** — not a finding: both are
  gitignored and untracked.
- **`services/providers/provider-priority.ts` vs `packages/core` duplicate** — not
  duplication: the services file is a thin snapshot builder over the core type.
- **`packages/config` / `packages/core` empty husks** — false: both have real content
  and consumers. (Only `packages/ui-cli` is a husk — plan 016.)
- **Mixed Bun/Node fs APIs; allmanga parity constants; `apps/cli/index.ts` wrapper** —
  by-design / already-tracked per CLAUDE.md; not findings.
- **TMDB public key literal at `bootstrap-persistence.ts:274`** — a distributed client
  key, not a private credential; out of scope.
