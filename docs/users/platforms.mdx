Kunai is Bun-first and terminal-first. Playback uses `mpv`; optional downloads use `yt-dlp`; `ffprobe` can validate completed artifacts when available. Poster previews use Kitty graphics or `chafa` depending on terminal capabilities.

**Who this is for:** anyone choosing an install path or optional tools for their OS.

**You will learn:** recommended installers per platform, optional dependencies, and platform-specific handoff behavior.

<Callout type="info" title="Optional tools stay optional">
Missing `yt-dlp`, `ffprobe`, or `chafa` should degrade gracefully ‚Äî setup and diagnostics explain the gap instead of breaking unrelated playback paths.
</Callout>

<Tabs items={["Linux", "macOS", "Windows", "Source checkout"]}>
<Tab value="Linux">

Recommended install (bundles Bun):

```sh
curl -fsSL https://raw.githubusercontent.com/KitsuneKode/kunai/main/install.sh | bash
kunai --setup
```

Package install alternative:

```sh
bun install -g @kitsunekode/kunai
kunai --setup
```

Recommended optional tools:

- `mpv` ‚Äî playback engine (required for watching)
- `yt-dlp` ‚Äî offline download jobs
- `ffprobe` ‚Äî optional validation of completed artifacts
- `chafa` ‚Äî poster previews outside Kitty-compatible terminals
- `xdg-open` ‚Äî `/docs`, `/report-issue`, folder reveal, and account auth handoffs

</Tab>
<Tab value="macOS">

Recommended install:

```sh
curl -fsSL https://raw.githubusercontent.com/KitsuneKode/kunai/main/install.sh | bash
kunai --setup
```

Package install alternative:

```sh
bun install -g @kitsunekode/kunai

```

Install `mpv` and optional download tools through Homebrew or your preferred package manager. Kunai uses `open` for browser and folder handoffs.

</Tab>
<Tab value="Windows">

Recommended install (PowerShell):

```powershell
irm https://raw.githubusercontent.com/KitsuneKode/kunai/main/install.ps1 | iex
kunai --setup
```

Kunai runs where a capable terminal and **mpv** are available. Windows Terminal is the most predictable surface.

Kunai uses `cmd /c start` for browser handoffs such as `/docs` and `/report-issue`. If an opener is missing, the shell should keep running and explain the limitation.

</Tab>
<Tab value="Source checkout">

```sh
bun install
bun run dev
bun run dev -- -S "Dune"
bun run dev -- -a -S "Attack on Titan"
```

Source checkouts are the best path for debugging provider changes, playback recovery, and documentation contributions.

</Tab>
</Tabs>

## Platform guardrails

- If an opener is missing, Kunai keeps running and leaves you in the current shell context.
- Provider and Discord live smokes are **manual** release checks ‚Äî not background activity triggered by normal use.
- Episode numbers are **1-based** in the UI; providers adapt internally.
