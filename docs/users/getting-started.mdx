Kunai is a terminal-first media shell. You search for a title, pick what to watch, Kunai resolves a direct-provider stream on your machine, and hands playback to `mpv`. The shell stays alive so you can recover, continue, or pick something else without restarting.

**Who this is for:** anyone installing Kunai for the first time.

**You will learn:** how to install, launch your first search, use high-signal shell commands, and recover when playback fails.

## Other install methods

The steps above use Bun global + `kunai --setup`. Use these tabs only when you need a different channel.

<Tabs items={["Binary", "npm global", "Source checkout"]}>
<Tab value="Binary">

Linux / macOS:

```sh
curl -fsSL https://raw.githubusercontent.com/KitsuneKode/kunai/main/install.sh | bash
kunai --setup
kunai -S "Dune"
```

Windows (PowerShell):

```powershell
irm https://raw.githubusercontent.com/KitsuneKode/kunai/main/install.ps1 | iex
kunai --setup
```

Release binaries embed Bun. You still need `mpv` for playback. Details: [Install and update](/docs/users/install-and-update).

</Tab>
<Tab value="npm global">

```sh
npm install -g @kitsunekode/kunai
kunai --setup
```

npm global installs may work, but Bun is what the project tests most often during beta.

</Tab>
<Tab value="Source checkout">

```sh
git clone https://github.com/KitsuneKode/kunai.git
cd kunai
bun install
bun run link:global
kunai --setup
```

Use `bun run dev -- <flags>` when you need to forward argv without a global link.

</Tab>
</Tabs>

## Launch recipes

Four common first-week patterns. Full flag list: [CLI reference](/docs/users/cli-reference).

### First play (search, then pick)

```sh
kunai -S "Dune"
```

`-S` opens search results — it does not auto-play. Add `--jump 1` or `-q` when you want the first match selected automatically.

### Anime mode

```sh
kunai -a -S "Attack on Titan" --jump 1
```

### Continue where you left off

```sh
kunai --continue
```

Use `--history` when you want to pick from recent entries instead of the newest unfinished one.

### Offline library

```sh
kunai --offline
```

`--zen --offline` keeps chrome minimal over the same library. Download-only bootstrap: `kunai --download -S "Title"` (resolve, queue, exit).

<Callout type="idea" title="Search does not mean auto-play">
`-S "Title"` lands on results. Discovery trays (`--discover`, `--random`, `--calendar`) also open without auto-play.
</Callout>

## In the shell

Press `/` to open the command palette. High-signal commands for a first week:

| Command | Purpose |
| --- | --- |
| `/history` | Recent watch progress |
| `/downloads` | Queued, running, and failed download jobs |
| `/library` or `/offline` | Playable completed downloads |
| `/diagnostics` | Runtime state and recent evidence |
| `/update` | Manual version check with install-method guidance |
| `/recover` | Refresh the current stream after failure evidence |
| `/fallback` | Try the next compatible provider |

## When playback fails

<Steps>
<Step>
### Stay in the shell
Do not restart blindly. Kunai keeps session context, provider inventory, and local watch history.
</Step>
<Step>
### Recover, then fallback
Run `/recover` to refresh the current provider intent. If that fails, `/fallback` walks the priority chain.
</Step>
<Step>
### Read evidence
Open `/diagnostics`. For a shareable bundle use `/export-diagnostics` — exports are redacted by default.
</Step>
</Steps>
