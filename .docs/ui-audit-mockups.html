🦊 Kunai — UI/UX Audit & Design Upgrade

Full surface audit across the new features (lists, playlist, stats, sync) plus cross-cutting issues. Each section has: root-cause analysis, before/after mockups, and implementation guidance.

Audit Findings Summary

Issues ranked by user-impact. Overflow bugs are now fixed (palette, stats). Remaining issues are design quality, engagement, and premium feel.

FIXED
Command palette overflows terminal — computePaletteMaxVisible was off by 6 rows. InputField=5 rows, palette chrome=4 rows. Now correctly computes all 25 rows of fixed overhead.
FIXED
Stats subtitle overflow → double-render — 9-line subtitle in ListShell + AppRoot header exceeded terminal height, causing wrap-around artifacts. Replaced with dedicated StatsShell.
HIGH
Stats heatmap missing entirely — formatHeatmap() existed but was never called. New StatsShell shows a colored GitHub-style heatmap with density depth.
HIGH
Watchlist/Playlist use plain list pickers — No visual hierarchy, no status affordances, no empty-state polish. Feels like a debug menu.
HIGH
No engagement hooks — Streak exists but is invisible until you open stats. No way to share, no social proof, nothing to come back to. Claude Code shows streak in header and celebrates milestones.
MED
Onboarding slide feels like a setup wizard — Lacks personality, the DotMatrix animation is functional but not "aha". No preview of the product you're about to use.
MED
Browse shell header padding wastes 2 rows — AppRoot paddingX + inner shell paddingX = 4 columns of wasted margin on each side. Only ListShell double-pads.
MED
Sync/connect flow has no visual feedback during OAuth — User is told to open browser but the shell shows nothing while waiting for the callback. Needs a spinner + countdown.

Implementation

Filter input should respect type:anime / type:series tokens.

Implementation

In handlePlaylist: use playlistService.getStatus() which returns { isStale, lastActivityAt, unplayedCount, nextItem }. Render two sections: "Up next" (unplayed, sorted by priority) and "Played" (dimmed, sorted by playedAt desc). Use previewGroup on ShellOptions to create group headers. Stale warning shows when isStale=true — rendered as a colored subtitle. r key calls playlistService.refillFromWatchlist(listService) and re-renders.

Implementation: Contextual idle state

In RootIdleShell (or via the BrowseShell empty state), render up to 2 lines of context when search query is empty and no results: (1) "Up next: [playlist.peekNext().title]" if playlist is non-empty, (2) today's calendar releases from catalogScheduleService. This gives the user something to do immediately on launch. Keep it hidden when results are showing.

Implementation: Header crumb enrichment

In root-status-summary.ts, buildRootStatusSummary already has streak and syncHealth. Extend the crumb field to include: watchlist count when non-zero, playlist unplayed count when non-zero. Format: "anime · vidking · 4wl · 3pl" or verbose when space allows. Keep it under 40 chars. The status header should show sync health as a colored badge: green ✓, amber ⚠, or dim ○ (disconnected).

Implementation: Engagement hook system

Milestones: In AppRoot useEffect, after streak is loaded compare to [3,7,14,21,30,50,100]. If crossing a threshold and config.lastStreakMilestone < threshold, show a transient alert for 8 seconds. Store in config.

Streak-at-risk: If streak ≥ 3, current time ≥ 20:00 local, and statsService.watchedToday() === false — show a single-line amber alert with the next playlist item. Once per day (check config.lastStreakRiskShownAt).

Share card: s key in StatsShell calls a platform-aware clipboard write. The text is a formatted card: ASCII heatmap + summary + show list + brand URL. Log to diagnosticsStore for analytics.
