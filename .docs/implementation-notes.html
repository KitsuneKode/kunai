Design decisions

  • 2026-05-19 Post-play footer per state — buildPostPlayFooterActions returns ≤4 actions keyed by postPlayState.kind instead of the old 15+ command dump. Rationale: launch spec §1.2; deeper actions stay behind /.
  • 2026-05-19 Details secondary shimmer — 32ms deferred resolve from cached browse data, not a network fetch yet. Rationale: instant primary zone; real schedule/TMDB can plug in without changing DetailsPaneUI API.
  • 2026-05-19 Claude Code–style density — prefer one context strip + one primary action; typography over bordered cards except amber “up next” emphasis (mockup-aligned).
  • 2026-05-19 Browse selection — amber ❯ prefix only (no teal row background); locked in implementation after mockup review.
  • 2026-05-19 Playing signal rail — right-aligned dim lines (res/fps, download speed, sub track); memory/health only after m in debug path.
  • 2026-05-19 Details companion poster — usePosterPreview inside DetailsPaneUI; left │ strip color = series state.

Deviations from spec

  • 2026-05-19 Post-play bordered cards — spec § typography-only; we keep a light borderStyle="round" on up-next/resume for scanability. Revisit if mockup strictness required.
  • 2026-05-19 Details secondary resolver — still cache/shim from browse option meta (airing/ended heuristics on previewMeta); no live schedule/TMDB fetch yet.
  • 2026-05-19 Playing footer “space pause” — label-only in footer; actual pause remains mpv-owned (footer action routes to command palette if activated).

Tradeoffs
